# AI1 Platform Schemas

## Purpose

This document defines the baseline JSON schemas called for in [plan.md](c:/Code/AI1-Platform/docs/plan.md). These schemas are the contract layer between the platform, workflows, and specialist agents.

The schemas are aligned to the current local-first architecture:

- Structured state is stored in `SQLite`
- Retrieval metadata may reference `ChromaDB`
- Large artifacts remain on the filesystem

## Scope

The canonical schemas in this document cover:

- `Story`
- `AcceptanceCriterion`
- `Task`
- `ArchitectureBlueprint`
- `ApiContract`
- `DataModelChange`
- `CodeChangeSet`
- `TestScenario`
- `AgentRunResult`
- `ReviewDecision`

Schema style:

- JSON Schema draft `2020-12`
- `additionalProperties: false` by default for top-level contracts
- IDs are stable strings generated by the platform

## Notes

- `Story` embeds acceptance criteria for intake and transport, while `AcceptanceCriterion` also exists as its own persisted entity.
- `Task.dependsOn` is included for agent handoff even though dependency links may also be normalized into a separate relational table.
- `CodeChangeSet` is included because [plan.md](c:/Code/AI1-Platform/docs/plan.md) lists it as a required shared contract even though it is not yet broken out in the current data model file.
- `ReviewDecision` includes `clarification` to match the decomposition ambiguity gate in the current workflow design.
